react特点:虚拟DOM(数据驱动)

自己开发框架思想:
1.数据(state)
2.模板(jsx语法)
3.数据 + 模板 => 原始的真实的DOM节点
4.数据变化(state change)
5.新的数据 + 模板 => 新的真实的DOM节点
6.新的真实的DOM节点 => 原始的真实的DOM节点

react开发思想:
1.数据(state)
2.模板(jsx语法)
3.数据 + 模板 => 原始的真实的DOM节点
4.根据原始的真实的DOM节点 => 原始的虚拟的DOM节点(一个用来描述DOM节点的JS对象)
5.数据变化(state change)
6.新的数据 + 模板 => 新的虚拟的DOM节点
7.把原始的虚拟的DOM节点和新的虚拟的DOM节点进行比较(diff算法)
8.(根据虚拟DOM节点的比较)替换原始的真实的DOM节点(只替换发生变化的部分)
